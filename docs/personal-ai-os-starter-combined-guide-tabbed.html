Personal AI OS Starter Guide

Installation Guide and Instruction Manual for a shareable Claude Code starter vault. Version 2026-05-31.

Installation Guide

Use this tab to install, import, personalize, secure, and back up the starter.

SectionAudienceInstructions / DetailsAction / ExampleSources
What this starter isAll usersPersonal AI OS Starter is a shareable Claude Code vault template for personal learning, portfolio building, research, writing, and structured knowledge work. It is not a finished personal assistant; it is a scaffold that asks the user to fill in their own profile, goals, privacy boundaries, learning style, and project priorities.Open START-HERE.md first, then run Claude Code in the starter folder and begin first-run onboarding.Claude Code Quickstart; Claude Code Memory
Why it was builtAll usersThe starter solves a common problem: people use AI chat tools in isolated conversations, then lose continuity. This vault turns Claude Code into a repeatable operating workspace where persistent instructions, templates, learning logs, project notes, and privacy rules live together.Use it when work spans multiple sessions: learning plans, portfolio projects, research notes, weekly reviews, writing drafts, or Git-backed documentation.Claude Code Memory; Claude Code Settings
What it is notAll users• Not a secure password manager. • Not a place to store government IDs, medical records, bank statements, raw secrets, or customer/company confidential data. • Not a replacement for judgment, source verification, or professional legal/medical/financial advice. • Not a fully autonomous system; users should review file edits, Git actions, and permission requests.Keep sensitive details outside the repo. Use placeholder files under private/ only, or a separate encrypted location.GitHub Ignoring Files; GitHub Secret Scanning; Claude Code Settings
PrerequisitesTechnical and non-technical users• A computer running macOS, Linux, Windows, or WSL. • A Claude account that can use Claude Code, or another supported Claude Code authentication route. • Terminal or command prompt access. • Git installed if the user wants backups and version history. • A GitHub account if the user wants a private remote backup. • Optional: VS Code, Cursor, or another editor.Non-technical users can still use the vault by following START-HERE.md and copying commands carefully.Claude Code Quickstart; GitHub Set up Git
Install Claude CodeAll usersInstall Claude Code using the official method for the operating system. Confirm installation by running claude --version or starting a session with claude.macOS/Linux/WSL: curl -fsSL https://claude.ai/install.sh | bash Homebrew alternative: brew install --cask claude-code Windows PowerShell: irm https://claude.ai/install.ps1 | iex Windows CMD: curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmdClaude Code Quickstart
Install or verify GitUsers who want backup/version controlGit tracks changes, creates safe checkpoints, and lets users back up the vault to a private GitHub repository. Users who are new to Git should learn status, add, commit, push, and pull before using advanced commands.Run: git --version. If unavailable, install Git using GitHub/Git official setup guidance for the OS.GitHub Set up Git
Unzip and place the starterAll usersUnzip the starter into a stable local folder. Avoid temporary locations such as Downloads if this will become a long-term vault.Recommended examples: macOS: ~/Documents/Personal AI OS Windows: C:\Users\<you>\Documents\Personal AI OS Linux: ~/Documents/personal-ai-osClaude Code Quickstart
Start Claude Code in the vault rootAll usersClaude Code should be started from the root folder that contains README.md, CLAUDE.md, AGENTS.md, and FIRST-RUN-ONBOARDING.md. This ensures project memory and rules are available.cd "/path/to/personal-ai-os-starter" claudeClaude Code Quickstart; Claude Code Memory
Run first-run onboardingAll usersThe starter is intentionally generic. First-run onboarding asks the user for their goals, learning context, preferred cadence, privacy boundaries, project interests, and writing/career use cases. It should create personalised starter files only after the user confirms the generated profile is accurate.Prompt: Read CLAUDE.md and FIRST-RUN-ONBOARDING.md. If onboarding is incomplete, guide me through the first-run setup. Do not read private/ or 99-archive/.Claude Code Memory; Claude Code Best Practices
Understand /memory outputAll usersWhen /memory is run, Claude Code may show project memory, imported AGENTS.md, rules, and user memory. For a shareable starter, users should keep personal operating rules in project files or user memory intentionally, not accidentally.Use /memory to inspect what is loaded. Do not add sensitive data to memory. Keep project CLAUDE.md concise and durable.Claude Code Memory
Review permissionsAll usersThe starter includes conservative .claude/settings.json rules. Users should keep destructive commands, Git write operations, web fetches, and sensitive-folder access as Ask or Deny unless they understand the consequences.Run /permissions. Recommended posture: normal file reads allowed when needed, Git push/commit ask-first, private folder deny, destructive shell commands deny.Claude Code Settings
Initialize Git safelyUsers who want backup/version historyBefore creating the first commit, users should review .gitignore, confirm sensitive folders are excluded, and run git status. Avoid git add . until the ignore rules have been reviewed.git init git status --short git add .gitignore git commit -m "chore: add gitignore safeguards"GitHub Set up Git; GitHub Ignoring Files
Configure Git identityUsers using GitHubGit commits need a name and email. Users who want privacy should use a GitHub noreply email and configure it at repository scope first, not necessarily globally.git config user.name "Your Name" git config user.email "YOUR_ID+YOUR_USERNAME@users.noreply.github.com"GitHub Set up Git
Create a private GitHub repoUsers who want remote backupCreate an empty private GitHub repository. If the local starter already has README.md and .gitignore, do not initialize the remote with another README, license, or .gitignore.Repository name example: personal-ai-os. Visibility: Private. Do not add README, .gitignore, or license if local files already exist.GitHub Pushing Commits; GitHub Non-fast-forward Errors
First push to GitHubUsers using GitHubAfter creating local commits and an empty private repo, add the remote and push. If push is rejected because the remote has existing work, inspect the remote before merging or force-with-lease pushing.git remote add origin https://github.com/YOUR_USERNAME/personal-ai-os.git git branch -M main git push -u origin mainGitHub Pushing Commits; GitHub Non-fast-forward Errors
Verify successful setupAll users• Claude Code starts from the vault root. • README.md, CLAUDE.md, and AGENTS.md are readable. • Onboarding prompts appear when onboarding is incomplete. • private/ is not used for real sensitive details. • Git status is clean after the first commit/push. • The GitHub repo is private if remote backup is used.Prompt: Run a setup sanity check. Do not read private/ or 99-archive/. Confirm memory, permissions, Git status, and next steps.Claude Code Memory; Claude Code Settings; GitHub Set up Git
Troubleshooting: claude not foundAll usersThe shell cannot find Claude Code. This usually means installation failed, the terminal has not been restarted, or the install location is not in PATH.Run claude doctor or reinstall using the official install method. Restart Terminal after installation.Claude Code Troubleshooting
Troubleshooting: wrong shell on WindowsWindows usersPowerShell and CMD use different install commands. Using the wrong command can produce confusing errors.If the prompt starts with PS C:\, use the PowerShell command. If it starts with C:\, use the CMD command.Claude Code Quickstart
Troubleshooting: fatal not a git repositoryGit usersThis means Git has not been initialized in the current folder, or the terminal is not inside the repository root.Run pwd or cd to the folder containing README.md. If needed, run git init only once in the correct root folder.GitHub Set up Git
Troubleshooting: push rejected / fetch firstGitHub usersThe remote repo has commits that local does not have. This often happens when GitHub created a README on the remote.Run git fetch origin and inspect git log origin/main --oneline before deciding whether to pull/rebase or use git push --force-with-lease.GitHub Non-fast-forward Errors; GitHub Pushing Commits
Troubleshooting: authentication failedGitHub usersGitHub may require browser authentication, GitHub CLI, a credential manager, SSH, or a personal access token depending on setup.Use GitHub’s official authentication guidance. If using a PAT, prefer fine-grained tokens and least privilege.GitHub Personal Access Tokens
Troubleshooting: Claude cannot read a fileAll usersThe settings file may deny the path, or the user may be asking Claude to read private/archive content. This is expected for sensitive areas.Check .claude/settings.json and /permissions. Do not weaken settings globally; adjust the request or explicitly authorize a narrow exception.Claude Code Settings
Troubleshooting: Claude ignores instructionsAll usersInstructions may be too long, contradictory, hidden in the wrong file, or not loaded because Claude Code was started from the wrong folder.Run /memory. Confirm CLAUDE.md and AGENTS.md are loaded. Keep core rules concise and move specialized rules into .claude/rules/.Claude Code Memory

Operating modelAll usersUse the starter as a living workspace. The core loop is: capture goals, plan learning, do hands-on work, produce artifacts, review progress, update the roadmap, and commit safe changes to Git.Weekly loop: plan 3 tasks, complete 1 artifact, update learning-log.md, run git status, commit safe changes.Claude Code Best Practices
When to use itAll users• When a task spans multiple days or sessions. • When you want persistent context instead of isolated AI chats. • When you are learning a technical topic through hands-on practice. • When building a portfolio artifact. • When drafting, revising, or organizing writing. • When maintaining a research or market-demand radar. • When preparing career narratives using sanitized information.Prompt: Based on my roadmap and project template, define the smallest useful portfolio project I can complete this week.Claude Code Best Practices; Claude Code Memory
When not to use itAll users• Do not use it as a secrets vault. • Do not paste raw passwords, API keys, government IDs, account numbers, or confidential third-party data. • Do not let Claude run destructive commands without review. • Do not store private personal records in Git. • Do not assume AI output is correct without source checking and validation.Use private placeholders and external secure storage. Ask Claude to summarize sensitive topics at a high level without recording exact identifiers.GitHub Secret Scanning; GitHub Ignoring Files; Claude Code Settings
First-run personalizationAll usersThe starter should prompt the user to create their own profile, goals, learning style, privacy boundaries, career context, project preferences, and review cadence. These should be stored as generic personal context files, not hard-coded into CLAUDE.md unless they are durable operating rules.Prompt: Guide me through first-run onboarding one section at a time. Ask before writing files. After each section, show me the proposed text.Claude Code Memory
Folder mapAll users• 00-core: identity, goals, learning style, operating principles. • 01-learning: roadmap, logs, weekly/monthly/quarterly reviews. • 02-technical-skills: topic-specific learning files. • 03-projects: portfolio and side-project structure. • 04-career: sanitized career positioning and interview prep. • 05-writing: writing drafts, blog ideas, style references. • 06-research: market radar and research briefs. • 07-personal-domains: non-sensitive personal domain planning. • private: placeholder-only sensitive domain structure. • 08-working: scratch work and active drafts. • 09-outputs: export-ready deliverables. • 10-sources: source summaries and references. • 11-templates: reusable templates.Ask Claude: Explain the folder structure and recommend where to put my first API learning project.Claude Code Memory
